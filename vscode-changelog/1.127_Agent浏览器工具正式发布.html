1.127 · Agent浏览器工具正式发布

2026年7月1日

查看发布说明 ↗
核心亮点Agent 窗口集成浏览器与工具其他重要变更
TL;DR

核心亮点

01

Agent 浏览器工具正式可用 (GA)

Agent 能直接打开网页、截图、点击验证自己的操作，无需外部 MCP 服务器，大幅提升自动化测试与 Web 应用构建效率。

02

每站点浏览器权限控制

集成浏览器支持摄像头、地理位置、设备等权限按站点提示授权，更安全地使用 Web API。

03

Agent 会话管理增强

支持自定义分组、拖拽排序、折叠分组，多会话窗口更有序。

04

Chat 输入横幅

CI 失败或 PR 评论有更新时，直接在聊天输入框上方显示操作按钮，无需离开对话。

05

子代理信用消耗透明化

悬停子代理区即可看到其消耗的 AI 信用额度，便于成本管理。

AGENT

Agent 窗口

  • 会话分组与拖拽：创建自定义组并折叠，支持拖拽调整会话顺序、移动会话到组或置顶区，批量选择后一起移动。
  • Chat 输入横幅：PR 的 CI 失败或新评论出现时，横幅提供“修复检查”和“查看检查”等快捷操作，无需切换上下文。
  • 编辑器沟槽反馈：审查 Agent 更改时，逐行悬停出现添加反馈图标，点击即可在指定位置留下评论，更快引导 Agent。
  • 更好的 PR 标题与描述：从 Agent 窗口创建 PR 时，自动利用会话上下文生成更准确的标题和描述。
  • 多聊天会话改进：同一 Agent 会话内可运行多个聊天：支持关闭/重开/永久删除聊天，所有聊天的进度与文件变更聚合显示，分支对话变成同会话的并行聊天。
  • 会话头部统一药丸按钮：工作区与变更药丸按钮样式统一紧凑，长名称截断，变更计数与多文件 diff 保持一致。
  • 切换会话时聚焦聊天输入：打开会话后键盘焦点自动落到聊天输入框，无需手动点击。
  • 响应式会话侧边栏（实验性）：窗口变窄时自动隐藏会话侧边栏，腾出编辑空间。
  • /troubleshoot 诊断 Agent 行为：在聊天输入中键入 /troubleshoot 后选择目标会话，即可分析日志并找出指令被忽略或响应缓慢的原因。
集成浏览

集成浏览器与工具

  • Agent 浏览器工具正式可用：Agent 可打开页面、读取内容与控制台错误、截图、选择/输入/导航，现已默认启用，支持企业策略全局禁用或限制域名。
  • 按站点权限管理：集成浏览器新增地理位置、摄像头/麦克风、传感器、剪贴板、设备（蓝牙/USB/串口/HID）等权限的弹窗询问，可在浏览器菜单中管理当前站点权限。
其他重要

其他重要变更

  • 终端命令沙箱（macOS/Linux）：Agent 调用的终端命令默认网络受限、文件系统受限，减少审批提示；需提权时才会请求批准。可通过权限下拉关闭。
  • 内建 Ollama 提供商废弃：推荐使用官方 VS Code 扩展，内建提供商已废弃。使用 BYOK 的用户需安装扩展并移除内建提供商以继续使用。
  • 子代理信用显示：悬停在子代理区即可查看该子任务消耗的 AI 信用额度，帮助成本管理。
  • 企业文件化 Copilot 设置：管理员可通过磁盘上的 managed-settings.json 文件（macOS/Linux/Windows 特定路径）下发 Copilot 策略，适用于未加入 MDM 的设备。
