核心亮点

01

GitHub Copilot 成为内置扩展

新用户无需单独安装扩展，即可使用 Copilot 聊天、内联建议和代理功能，降低使用门槛。

02

Agent 调试日志支持历史会话

Agent Debug Logs 面板可查看当前和过去的 agent 会话日志，日志持久化至磁盘，便于排查自定义交互问题。

03

Copilot CLI 配置思考努力

在 Copilot CLI 中为推理模型选择思考努力级别（thinking effort），平衡响应质量与延迟。

04

终端 agent 工具支持前台终端

send_to_terminal 和 get_terminal_output 工具可作用于任意可见终端（如 REPL），agent 能直接读写交互式进程。

05

无障碍支持大幅提升

Agents App 新增无障碍对话框、键盘导航命令和 ARIA 标注，屏幕阅读器用户可高效导航。

AI

AI Copilot 与 Agent 体验

  • Customizations 欢迎页面：Chat: Open Customizations 命令打开时显示引导页，支持通过自然语言描述自动生成 agent、技能和指令。
  • 工具确认轮播（实验性）：多工具调用时以轮播方式紧凑展示，无需滚动即可逐一审批，由设置 chat.tools.confirmationCarousel.enabled 控制。
  • JS/TS Chat Features 扩展（预览）：内置扩展增强 Copilot 对 TypeScript/JavaScript 的技能，目前支持创建现代 TypeScript 项目，需启用 jsts-chat-features.skills.enabled。
  • 代理网络访问过滤（企业）：管理员可通过组策略限制 agent 工具（fetch、集成浏览器等）的网络域名访问，支持白名单和黑名单。
Terminal

终端与调试

  • 终端输入改进：移除了基于 LLM 的输入检测，减少延迟和 token 消耗；agent 通过 send_to_terminal 直接处理，并提供 Focus Terminal 按钮。
  • 背景终端通知默认启用：agent 在背景终端运行命令时自动接收完成、超时或需输入的通知，无需轮询，响应更及时，设置 chat.tools.terminal.backgroundNotifications 默认 true。
  • Agent 调试日志面板：支持查看当前和历史会话日志，日志持久化至本地磁盘，设置 github.copilot.chat.agentDebugLog.fileLogging.enabled 开启。
UI

UI/UX 与其他改进

  • Diffs 直接嵌入聊天：代码差异对比直接在聊天会话中渲染，无需切换至独立 diff 视图，减少上下文切换。
  • 聊天渲染性能优化：流式渲染更高效，减少布局抖动，修复快速工具调用导致扩展主机挂起的问题。
  • 集成浏览器新入口：View > Browser 菜单和快捷键 ⌥⌘/ (Win/Linux Alt+cmd+/) 可快速打开集成浏览器。
  • Edit Mode 正式弃用：从 1.126 起完全移除 Edit Mode，当前可通过 chat.editMode.hidden 临时启用。
  • GitHub Pull Requests 扩展更新：新增创建 PR 的聊天工具，支持从删除本地分支命令删除工作树。
