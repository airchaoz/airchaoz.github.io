核心亮点编辑器源代码管理调试与终端
TL;DR

核心亮点

01

Diff 编辑器支持自动换行

内联和并排视图现在都支持单词换行，并新增 diffEditor.wordWrap 设置独立控制。

02

扩展二分查找（Extension Bisect）

通过 Help: Start Extension Bisect 命令，利用二分算法快速定位引发问题的扩展。

03

键盘快捷键编辑器改进

现在可以直接从命令面板的齿轮按钮为命令绑定快捷键，并支持多键绑定和更丰富的搜索过滤。

04

文件资源管理器撤销/重做

支持对删除、重命名、复制、移动及新建文件/文件夹等操作进行 Undo/Redo，与编辑器撤销栈独立。

05

预览编辑器交互优化

默认关闭从快速打开进入预览模式；代码导航时起始编辑器自动退出预览；新增 Keep Editors Open 菜单项。

06

空格缩进的粘滞制表符

新设置 editor.stickyTabStops 让光标在缩进空格中移动时行为类似制表符。

Editor

编辑器

  • 单词补全支持跨文件：editor.wordBasedSuggestionsMode 可设为 allDocuments 或 matchingDocuments，从其他打开文件提取建议。
  • CodeLens 自定义字体和大小：新增 editor.codeLensFontFamily 和 editor.codeLensFontSize 设置，可调整 CodeLens 外观。
  • 删除整个单词命令：新增 Delete Word 命令，一次性删除光标所在的整个单词。
  • 内联建议详情可隐藏：editor.suggest.showInlineDetails 设为 false 可关闭建议列表内联详情，节省空间。
  • TypeScript 自动导入补全显示路径：当存在同名符号时，补全项会标注导入路径以帮助区分。
SCM

源代码管理

  • Git 新命令：添加 Git: Cherry Pick、Rename、Push Tags、Checkout to (Detached) 等命令。
  • Git 新增设置：支持 pruneOnFetch、ignoreSubmodules、openAfterClone、followTagsWhenSync 等。
  • 文件操作撤销支持事件：Explorer 的创建/重命名/删除等操作现在支持撤销，且会触发相应生命周期事件。
  • Source Control 视图查看父文件夹路径：当标签页禁用时，diff 编辑器现在显示文件的父文件夹路径而非仅文件名。
  • 删除文件合并冲突优化：改进删除文件类型合并冲突的 diff 显示，展示另一分支的实际变更。
Terminal

调试与终端

  • 异常断点支持条件：可在断点视图中为异常断点设置条件（仅 Mock Debug 示例，后续扩展将跟进）。
  • 调试行显示在概览标尺：当前调试行和焦点调试行分别用 editor.stackFrameHighlightBackground 和 editor.focusedStackFrameHighlightBackground 高亮。
  • 自动调试配置持久化：最近使用的自动调试配置会在重启后保留，失效时会提示重新选择。
  • 调试控制台折叠重复行：重复输出内容自动折叠并显示出现次数。
  • 终端配置入口：终端下拉菜单新增 Configure Terminal Settings 选项，可直接修改终端相关设置。
  • 环境变量处理改进（Linux/macOS）：首次从 Dock/启动器启动时，后台解析 shell 环境（超时 10 秒有提示），也支持终端内变量传递。
