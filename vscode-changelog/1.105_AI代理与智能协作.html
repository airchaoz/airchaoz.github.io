TL;DR

核心亮点

01

内置规划代理 (Plan Agent)

可分析复杂任务、分解步骤并生成实施计划，支持将计划直接启动实现或保存为后续使用，帮助避免遗漏关键需求。

02

聊天模式过渡 (Handoffs)

允许在自定义聊天模式间切换并携带上下文和预设提示，便于编排多步骤开发工作流，例如从规划过渡到实现。

03

隔离子代理 (Subagents)

子代理拥有独立上下文窗口，自动执行研究、分析等任务后将结果返回主会话，优化上下文管理，避免主会话被无关信息污染。

04

OpenAI Codex 与 Copilot CLI 集成

Agent Sessions 视图统一管理本地和云端代理会话，支持直接与 OpenAI Codex 及 GitHub Copilot CLI 协作，可跨代理委托任务。

05

AI 合并冲突解决

打开含冲突标记的文件后，编辑器右下角出现新操作按钮，可启动 AI 流程自动分析两边分支并合并基线，生成解决建议。

06

OS 通知增强

任务完成和聊天响应时显示系统级通知与徽标，包含回复预览，点击可直接聚焦相关窗口。

AI 与聊天

  • 全限定工具名 (Fully Qualified Tool Names)：在提示文件与自定义聊天模式中使用 search/codebase 等限定名避免命名冲突，并支持代码操作协助迁移。
  • 嵌套 AGENTS.md 文件 (实验性)：支持在子文件夹中放置嵌套 AGENTS.md 文件，为不同代码部分提供针对性指令，需设置 chat.useNestedAgentsMdFiles 启用。
  • 聊天用户体验改进：OS 通知、思维链显示 (GPT-5-Codex 的思考过程)、最近本地会话历史及在代理循环中即时保留/撤销文件编辑。
  • MCP 市场预览 & 自动启动：Extensions 视图中内置 MCP 市场，可浏览安装 MCP 服务器；新/过时服务器在发送聊天消息时自动启动，避免干扰性弹窗。
  • MCP 资源展示改进：工具结果中包含资源预览与完整获取指令，提升模型使用工具的准确率。
  • 聊天终端配置文件：新增平台特定设置 (chat.tools.terminal.terminalProfile.*) 可为聊天代理指定专用 shell，简化交互元素不影响主环境。
  • 终端自动回复提示 (实验性)：启用 chat.tools.terminal.autoReplyToPrompts 后可让代理自动应答终端 y/n 确认，减少手动干预。
  • 快捷键导航历史消息：新增 Alt+Cmd+Up/Down 快捷键快速遍历聊天输入历史。
