TL;DR

核心亮点

01

只读模式

通过 `files.readonlyInclude` / `files.readonlyExclude` 设置，将工作区中的特定文件或文件夹标记为只读，防止误修改（如 node_modules）。

02

“粘贴为”选项

粘贴文件后出现选择控件，可切换为绝对路径、相对路径、Markdown 链接或附件，并支持通过 `Paste As...` 命令预选。

03

自动复制外部文件到工作区

拖放或粘贴图片、视频、音频到 Markdown 文档时，自动将文件复制到工作区并插入引用，可配置目标路径和覆盖行为。

04

默认 Git 分支名改为 main

新建 Git 仓库默认分支为 main，可通过 `git.defaultBranchName` 配置自定义。

05

笔记本富内容搜索

在搜索控件中可搜索笔记本输出内容（如文本、图像），并通过切换过滤搜索输入/输出/代码等单元类型。

06

JSX 标签关联编辑

启用 `editor.linkedEditing` 后，修改 JSX 开始标签自动更新匹配的结束标签，并支持 F2 重命名。

Editor

编辑器 & 粘贴为 & Markdown

  • “粘贴为”控件增强：粘贴后显示小控件（可通过 ⌘. 打开），还可设置 editor.pasteAs.showPasteSelector 为 never 禁用。
  • 只读模式会话覆盖：提供命令临时切换当前编辑器的只读/可写状态，覆盖设置配置。
  • 自动复制外部文件配置：markdown.copyFiles.destination 支持 glob 和变量定制目标路径；markdown.copyFiles.overwriteBehavior 控制覆盖策略。
  • Markdown 中 HTML 路径的 IntelliSense：Markdown 文件中 HTML 标签内的文件路径也支持补全、引用查找、重命名和验证。
  • 粘贴音频插入 ：拖放或粘贴音频文件到 Markdown 文档自动插入 HTML5
  • JSX 标签 F2 重命名：在 JSX 标签上按 F2 仅重命名匹配标签（类似 HTML），可通过 javascript.preferences.renameMatchingJsxTags 配置。
Terminal

终端 & 源代码管理

  • Fish shell 自动集成：终端 Shell 集成自动为 fish shell 启用，增强命令历史、提示等体验。
  • 终端 overline 支持：支持 ECMA-48 上划线转义序列（SGR 53/55），常用于终端底部的状态栏。
  • 默认分支名配置：新建仓库默认分支改为 main，也可通过 git.defaultBranchName 自定义。
  • 分支选择器集成 vscode.dev 与 GitHub：分支选择器下拉中每个分支旁显示“在 vscode.dev 上签出”或“在 GitHub 上打开”按钮。
  • 相似度阈值设置：git.similarityThreshold 配置 Git 重命名检测的相似度阈值（0-100，默认50）。

笔记本、语言与 Copilot Chat

  • 笔记本搜索过滤：搜索控件新增切换过滤，可限制仅在代码、输出或标记单元格中搜索。
  • 笔记本格式运行：新增 notebook.formatOnCellExecution 设置在每次运行单元格时自动格式化。
  • 笔记本代码操作保存：支持 notebook.codeActionsOnSave 在保存笔记本时运行代码操作。
  • TypeScript 5.1：内置 TypeScript 5.1.3，带来新语言特性与性能改进。
  • JSDoc @param 补全：编写 JSDoc 注释时自动建议所有缺失的 @param 参数，并创建类型占位符。
  • JSON Lines 语法高亮：文件扩展名 .jsonl 现在获得语法高亮支持。
  • Copilot Chat 会话管理：在稳定版 VS Code 中可用；支持保存/浏览会话历史、导出/导入为 JSON、删除对话条目、将聊天从侧边栏移到编辑器，以及内联对话的 live preview 模式。
  • Copilot Quick Question 实验：启用 chat.experimental.quickQuestion.enable 后，可通过快捷键快速提问，无需离开上下文。
