核心亮点

01

专用的 Agent Host

代理会话在独立进程中运行，支持从多个 VS Code 窗口同时连接和渲染，为 Copilot、Claude、Codex 等提供统一基础设施。

02

Agents 窗口新编辑器面板（实验性）

审查 agent 生成的文件和 diff 时不再切换面板，直接在 docked 编辑器中打开，支持 inline/side-by-side diff 切换、紧凑 diff 视图和编辑快捷键。

03

以 ! 前缀运行命令

在聊天输入中以 ! 开头即可直接执行终端命令，无需离开对话窗口。

04

现代 UI 预览（实验性）

启用 workbench.experimental.modernUI 即可预览更新后的 VS Code 工作区外观，Insiders 版本默认开启。

代理与聊

代理与聊天

  • Agent Host 架构详解：代理会话运行在独立进程中，基于 Agent Host Protocol (AHP)，同一会话可被多个窗口连接。Copilot 代理由 Copilot SDK 驱动，行为与 Copilot CLI 等产品一致。启用 chat.agentHost.enabled 后从 harness 下拉菜单选择。
  • 会话管理工具：运行在 Agent Host 上的代理可枚举、创建、观察和操作其他会话，例如列出所有会话、读取对话历史、创建子会话并发送消息。VS Code 会显示跳转链接，发送消息前需用户确认。
  • Agents 窗口改进：新建会话时记住上次的代理模式和审批选择；用单个「New Worktree」复选框替代文件夹/工作树下拉菜单，简化隔离设置。
  • BYOK 模型支持：在 Agents 窗口中选择 Copilot harness 并运行在 Agent Host 上时，可使用自带密钥 (BYOK) 模型。
  • 迁移 Prompt 文件到 Skills（实验性）：启用 chat.customizations.promptMigration.enabled 后，可将 .prompt.md 文件迁移为 skills，实现跨 harness 兼容。
Editor

编辑器与现代 UI

  • 从工具栏重新打开编辑器：当文件或 diff 支持多个编辑器时，通过编辑器工具栏的 ... 菜单 → Reopen Editor With 即可直接切换，避免使用命令面板。
  • 现代 UI 预览：实验性设置 workbench.experimental.modernUI 更新整个工作区的视觉风格，目前 Insiders 默认启用。
其他特性

其他特性

  • GitHub Enterprise 支持 Copilot 认证：当 Copilot 通过 GHE 实例提供时，Agent Host 支持完成 OAuth 和令牌请求，适用于编辑器窗口和 Agents 窗口中的 Copilot 与 Claude 代理。
  • 自定义编辑器优先级 API（提议）：customEditorPriority 新 API 允许为文件、diff 和 merge 编辑器分别设置优先级（default/option/never），diff 和 merge 默认优先使用内置文本编辑器。
