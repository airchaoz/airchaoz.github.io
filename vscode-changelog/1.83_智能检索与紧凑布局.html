核心亮点

01

命令面板相似命令列表

在命令面板新增“相似命令”区域，帮助查找记忆模糊的指令，即使拼写不精确也能被推荐，大幅降低命令检索难度。

02

配置文件图标自定义

可为每个配置文件关联图标，活动栏上直接显示当前激活的配置图标，多配置切换一目了然。

03

紧凑编辑器标签高度

新增 window.density.editorTabHeight 设置，缩小标签垂直空间，为代码编辑区域腾出更多可视面积。

04

固定标签独立行支持拖拽

通过 workbench.editor.pinnedTabsOnSeparateRow 可将固定标签置于独立行，并且在标签行之间拖拽即可自动固定或取消固定。

05

笔记本中“转到符号”与滚动配置

启用 notebook.gotoSymbols.showAllSymbols 后可在笔记本内快速定位符号；新增 notebook.scrolling.revealNextCellOnExecute 精确控制执行下一单元格时的滚动行为。

Editor

编辑器与工作台

  • 自动保存时执行 Code Actions：editor.codeActionsOnSave 支持 always 值，可在显式保存及窗口/焦点变更的自动保存时触发代码操作，进一步提升代码清理自动化。
  • 设置编辑器搜索结果优化：搜索结果减少噪音，优先展示常用设置，并可配置 workbench.settings.settingsSearchTocBehavior 恢复旧有排序。
  • 主题选择器显示主题标识符：颜色主题选择器现在同时显示本地化名称和英文标识符，方便多语言用户准确识别。
  • 编辑器操作按钮溢出优化：当操作按钮过多时，关闭、拆分等关键按钮始终可见，其余折入“更多操作”菜单。
笔记本

笔记本

  • 空白字符相关设置生效：笔记本文件现遵循 files.trimTrailingWhitespace、files.trimFinalNewlines 和 files.insertFinalNewline 等保存时的空白处理规则。
  • 最终化笔记本 Code Action API：笔记本现完全支持 contributed Code Actions，并引入 notebook.codeActionsOnSave 设置，允许对整个笔记本运行源码操作（kind 前缀 notebook.source.）。

语言、调试与扩展生态

  • Perl 6 更名为 Raku：语言标识符由 perl6 改为 raku，.raku 文件自动关联 Raku 语言。
  • JavaScript 调试器支持 WebAssembly 调试：结合 WebAssembly DWARF Debugging 扩展，可调试编译为 WebAssembly 的代码（如 Emscripten 编译的 C++），并在调用堆栈中显示方法/函数上下文。
  • Python Debugger 扩展与 justMyCode 全局设置：Debugpy 扩展更名为 Python Debugger，新增 debugpy.debugJustMyCode 用户设置，全局控制是否步入系统/第三方库代码，无需逐项目配置 launch.json。
  • Pylint 即时检查与 Mypy 范围控制：Pylint 新增 pylint.lintOnChange 在输入时即时报告错误；Mypy 扩展支持 reportingScope（file/workspace）和 preferDaemon（使用 dmypy）加速类型检查。
  • Pylance 调用参数嵌入提示细粒度控制：python.analysis.inlayHints.callArgumentNames 现支持 partial（仅常用参数）、all（全部）或 off，更灵活显示函数调用时的参数名提示。
  • GitHub Copilot 测试生成与内联多轮对话：/tests 改进能自动检测测试框架并生成风格一致的测试代码；内联聊天保留多轮上下文，可引用此前消息。
  • Jupyter 服务端 API 最终化与 JupyterHub 扩展：Jupyter 扩展提供正式 API 供第三方贡献自定义服务器；新增 JupyterHub 扩展支持登录 JupyterHub 并直接在 VS Code 中运行内核。
