IntelliSense in chat responses

Copilot 生成的代码块支持跳转定义、悬停提示、转到实现等 IntelliSense 功能，帮助快速理解 AI 输出。

WORKBENCH

Workbench

  • Editor tabs multi-select：批量操作多个编辑器标签。使用 Ctrl+Click（macOS Cmd+Click）添加选中，Shift+Click 范围选中。
  • Always Show Editor Actions：设置 editor.alwaysShowEditorActions 为 true 后，无论编辑器是否激活，标题栏操作按钮始终显示。
  • Custom profile for new window：通过 window.newWindowProfile 指定新窗口使用的 profile，不再默认继承激活窗口或默认 profile。
COPILOT

Copilot & AI

  • Chat & Language Model API 正式化：Chat Participants API 允许扩展以 @ 参与对话；Language Model API 提供对 GPT-3.5/4 的访问，需用户授权。
  • Attach more context to chat：现在可附加工作区符号、文件等更多上下文；支持 📎 图标或 ⌘/（Windows/Linux Ctrl+/）调出选择器。
  • IntelliSense in chat code blocks：Copilot 回答中的代码块支持跳转、悬停、转到定义等 IntelliSense，默认支持 TypeScript/JavaScript/HTML/CSS。
  • Automatic rename suggestions：重命名符号时自动触发 Copilot 重命名建议，可通过 github.copilot.renameSuggestions.triggerAutomatically 关闭。
  • VS Code Speech: text-to-speech for Copilot responses：启用 accessibility.voice.autoSynthesize 后，语音输入的回答自动朗读；也可通过扬声器图标逐条播放。
EDITOR

Editor & Terminal

  • Notebook: Find in cell selection：设置 notebook.experimental.find.scope.enabled 为 true 后，可在选定单元格区域内搜索，提高笔记本内查找精准度。
  • Terminal: glyph rescaling enabled by default：terminal.integrated.rescaleOverlappingGlyphs 默认开启，自动缩放宽度不明确的字符（如罗马数字），避免重叠。
  • Terminal: canvas renderer removed：canvas 渲染器已完全移除，不支持 WebGL2 的机器自动回退至 DOM 渲染器，请确保 GPU 加速兼容。
