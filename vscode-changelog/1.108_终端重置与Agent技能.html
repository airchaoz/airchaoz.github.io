核心亮点

01

Agent Skills 实验性功能

允许通过 SKILL.md 定义指令和脚本，让 GitHub Copilot 在上下文相关时自动加载执行专有任务，大幅扩展 Copilot 的领域能力。

02

终端 IntelliSense 默认 UX 重做

默认改为仅显式触发（Ctrl+Space），避免打断重度用户的肌肉记忆，同时提供状态栏快速切换和一次性提示，兼顾可发现与不侵扰。

03

终端性能与稳定性大提升

macOS/Linux 大文本粘贴不再节流，修复了布局抖动、CommitMono 字体崩溃等影响编辑器流畅度的关键问题。

04

终端自定义字形覆盖近 800 个字符

GPU 渲染器现在支持 Braille、Powerline、Git 分支符号等完整字符集，无需依赖字体，对齐更精准。

05

拖拽导入配置文件

将 .code-profile 文件拖入 VS Code 即可预览并导入设置，分享与切换环境更便捷。

06

Git blame 可忽略空白变更

新增 git.blame.ignoreWhitespace 设置，帮助开发者聚焦于实际功能变更的提交，而非代码格式化带来的噪音。

Editor

编辑器与源码管理

  • 复制面包屑路径：新增“Copy Breadcrumbs Path”命令，支持自定义分隔符（breadcrumbs.symbolPathSeparator），方便分享符号位置。
  • 符号搜索支持特殊字符：工作区符号搜索（⌘T）不再因为 # 等字符过滤所有结果，适配 rust-analyzer 的修饰符用法。
  • 代码片段新转换：snakecase / kebabcase：两种新变换应用于片段转换，例如 ${TM_FILENAME/(.*)/${1:/snakecase}/} 将文件名转为蛇形命名。

  • 提交消息编辑器改进：提交/取消操作按钮移至编辑器右下角覆盖控件，更易发现。
  • 工作树（Worktrees）实验性支持：源代码管理仓库视图中新增工作树节点，支持打开、删除等操作，需启用 scm.repositories.explorer 设置。

终端与AI协同

  • 终端 IntelliSense 参数提示改进：状态栏改为图标+滚轮切换选择模式；新增眼睛图标可一键重新开启快速建议；启动时新提示说明如何显示补全。
  • 终端尺寸调整显示覆盖层：调整大小时短暂显示当前行列数（列 x 行），灵感来自 ghostty。

  • 终端工具自动批准规则扩展：默认自动批准 git、rg、sed 等安全子命令，以及 package.json 中通过 npm/pnpm/yarn 运行的脚本，并新增会话/工作区粒度授权。
  • 终端工具命令不再进入 shell 历史：启用 shell 集成后，终端工具运行的命令自动以空格开头（bash）等方式屏蔽，防止历史污染。
  • Agent Sessions 视图改进：支持键盘操作、按状态/时间分组、显示更改文件与 PR、批量归档；聊天选择器现在基于会话信息。
  • 聊天标题栏显示优化及空面板恢复：聊天标题始终在活动栏之外显示；重启后默认显示空聊天面板，可通过 chat.restoreLastPanelSession 恢复旧行为。
