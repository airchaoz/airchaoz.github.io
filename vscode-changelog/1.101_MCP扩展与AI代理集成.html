TL;DR

核心亮点

01

MCP 全面升级：Prompts / Resources / Sampling / Auth

MCP 服务器现支持 prompts（可复用聊天片段）、resources（资源浏览与附加）、sampling（服务器回调模型）以及 OAuth 认证，极大拓展 Agent 工作流能力。

02

Chat 工具集与自定义模式

可创建工具集分组管理相关 tools，并支持定义自定义 Chat 模式（如只读规划模式），按需配置指令与可用工具。

03

Copilot Coding Agent 内嵌到源码管理

GitHub PR 扩展支持直接在 VS Code 中分配 Issue/PR 给 Copilot，追踪其工作状态，形成闭环任务流。

04

Notebook 中 Agent 执行跟随模式

开启后 Notebook 视图自动滚动到被 Agent 执行的 Cell，实时跟随迭代，并可随时关闭。

05

终端语言服务器补全（Python 先行）

终端交互式 Python REPL 会话中获得 Pylance 提供的语言补全，与编辑器体验一致。

AI

AI & Chat 改进

  • MCP 认证与调试模式：支持需要 OAuth 的 MCP 服务器，提供 Development 模式（watch + debug），可本地调试 MCP 服务器。
  • 隐式上下文与任务诊断感知：当前文件自动作为建议上下文；Agent 运行时能识别 Task 配置中的错误并建议修复。
  • 浮动窗口增强：浮动 Chat 窗口标题栏新增“停靠回去”和“开启新会话”按钮。
  • Fetch 工具注入警告：使用 fetch 获取网页时增加提示注入风险警告。
Editor

编辑器与终端体验

  • Find as you type 可配置：新增 editor.find.findOnType 设置，可关闭即时搜索，改为按 Enter 触发。
  • EditContext API 默认启用：editor.editContext 设置默认开启，修复 IME 相关 Bug，为未来输入体验改进铺路。
  • 设置 AI 搜索建议：Settings 编辑器新增 AI 搜索开关，支持语义匹配（如搜“增大字号”找到 font.size）。
  • 搜索关键词建议性能提升：Search 视图中的关键词建议响应速度提升约 5 倍，设置已迁移至 core。
  • 终端 Shell 环境发现（Windows PowerShell）：VS Code 现在可继承 PowerShell 配置的环境变量（如 PATH）。
SCM

源代码管理与 Notebook

  • 源码管理历史项详情：在 Source Control Graph 视图中点击条目可查看资源树/列表，支持多文件 diff 打开。
  • 历史项加入 Chat 上下文：支持将某个 commit 或 PR 作为上下文添加到 Chat 询问中。
  • Notebook 工具与长流程：Jupyter 扩展贡献 Notebook 配置工具（内核选择、虚拟环境），Agent 能自动维持长上下文。
  • Notebook Cell 运行确认预览：Agent 请求运行 Cell 时，在确认对话框中显示代码片段，并可直接导航到 Cell。
