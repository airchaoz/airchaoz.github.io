代理与工

代理与工作流

  • 代理会话集成到聊天视图：代理会话列表直接嵌入 Chat 视图，支持紧凑、侧边栏两种布局，并持久化筛选与切换状态。
  • 本地代理后台持续运行：关闭聊天会话后，本地代理任务继续执行，可在会话列表查看进度并随时切回。
  • 为后台代理添加上下文：支持将选中内容、问题、符号、Git 提交等作为上下文附加到后台代理提示，构建更精准的指令。
  • 组织级自定义代理（实验性）：允许在 GitHub 组织层面定义自定义代理，成员可在聊天中直接使用组织级代理。
  • 自定义代理可作子代理（实验性）：复杂任务中主代理可委托子代理独立处理子议题，自定义代理也可被用作子代理。
  • 复用 Claude Skills（实验性）：支持加载 ~/.claude/skills 或项目中的 Claude Skills，作为按需能力供代理使用。

聊天与 AI 体验

  • 内联聊天优化为代码编辑：内联聊天聚焦于当前文件的快速修改，无法处理的任务自动升级至 Chat 视图并保留上下文。
  • 语言模型集中管理：新增 Language Models 编辑器，统一查看和管理所有可用的语言模型，支持搜索、过滤、开关可见性。
  • URL 与域名自动审批增强：fetch 工具新增两步审批流程：先批准域名访问，再批准内容注入聊天，防范提示注入攻击。
  • fetch 工具支持动态内容：现在可正确获取由 JavaScript 渲染的网页内容（如 SPA、Jira 页面），不再返回空结果。
  • 文本搜索可查忽略文件：#textSearch 工具支持搜索 .gitignore 或 files.exclude 指定的文件（如 node_modules），并提供提示以启用搜索。
  • 聊天终端输出丰富化：Terminal 响应中的输出现在以完整只读 xterm 终端形式渲染，保留 ANSI 颜色和运行时间信息。
  • 敏感文件差异预览：对 settings.json 等敏感文件的编辑，现在显示差异视图而非原始文本，便于审批。
  • 推理与工具调用可折叠：模型推理文本和连续工具调用默认折叠，以 chat.agent.thinkingStyle 和 chat.agent.thinking.collapsedTools 设置控制。
