核心亮点

01

Profiles 正式发布

可创建包含扩展、设置、快捷键、UI状态等在内的配置文件，并支持导出/导入分享，针对不同开发场景快速切换工作环境。

02

VS Marketplace 扩展签名

所有上传至 VS Marketplace 的扩展默认进行代码签名，安装时验证签名确保来源可信且未被篡改，提升扩展生态安全性。

03

终端可访问性升级

新增终端屏幕阅读器模式（Terminal: Focus Accessible Buffer）和终端可访问性帮助命令，改善视障用户使用终端的体验。

04

多视图拖拽缩放

支持拖拽视图角落同时调整多个视图的大小，简化布局操作。

05

终端链接检测增强

现在可检测包含空格、方括号、各类行列格式的链接，并支持更多协议（如 vscode://、file://），大幅提升终端交互效率。

06

新增 Git 命令

支持仅暂存已暂存的更改（Git: Stash Staged）以及删除远程标签（Git: Delete Remote Tag），完善 Git 工作流。

Editor

工作台与编辑器

  • 网格布局保持最小化状态：当编辑器窗口被最小化时，调整整体工作区或侧边栏尺寸可保持该宽度不变，避免布局意外改变。
  • 树形视图搜索历史：树形视图的搜索控件支持上下箭头浏览历史搜索词，方便快速重新执行之前的搜索。
  • 键盘快捷键编辑器增强：显示贡献快捷键的扩展名称，并支持在 when 子句中通过 Ctrl+Space 触发上下文键建议。
  • 建议选择模式可配置：新增 editor.suggest.selectionMode 设置，可控制自动建议是否自动选中最佳项，避免误触。
  • 代码操作列表可滚动：当扩展提供的代码操作过多时，现在支持在区域内滚动查看，不再被截断。
  • 跳转到特定匹配项：在查找控件中新增“Go To Match...”命令，根据匹配编号直接跳转到指定位置。
  • 内联建议工具栏重新设计：工具栏更紧凑，提供快速循环建议和逐词接受的功能，并新增默认快捷键（Ctrl+左右箭头）。
  • 打开大文件前确认：新增确认对话框以防止意外打开超大文件（尤其远程环境），可通过 workbench.editorLargeFileConfirmation 调整阈值。
Terminal

终端与 Git

  • 终端新默认快捷键：新增打开检测链接（Ctrl/Cmd+Shift+O）、转到最近目录（Ctrl/Cmd+G）、运行最近命令（Ctrl+Alt+R）等快捷键，提升终端操作效率。
  • Powerline 三角形/对角线自定义字形：GPU 加速终端中为 Powerline 额外符号提供像素完美渲染，避免字体宽度歧义。
  • 括号粘贴模式支持：“在活动终端中运行选中文本”命令将使用括号粘贴模式，多行选择作为单一输入执行，减少错误。
  • Git 命令在执行中禁用：在 Checkout、Commit、Push、Pull 操作进行时，禁用大多数 Git 命令，防止因并发操作导致数据丢失。
  • 父文件夹 Git 仓库处理优化：不再自动打开工作区/文件父文件夹中的 Git 仓库，改为通知并引导用户手动选择，避免误操作数据丢失。
  • Cygwin/MSYS2 不安全配置文件检测：重新检测 Cygwin、Cmder、MSYS2 等 shell，但要求用户通过“选择默认配置”显式确认后才启用，避免安全风险。
