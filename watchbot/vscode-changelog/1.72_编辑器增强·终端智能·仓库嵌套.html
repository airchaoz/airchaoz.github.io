1.72 · 编辑器增强·终端智能·仓库嵌套

查看发布说明 ↗
核心亮点工作台与编辑器源代码管理与终端语言与扩展
TL;DR

核心亮点

01

工具栏自定义

右键隐藏/显示工具栏操作，隐藏项移至“更多操作”菜单，可重置菜单恢复。

02

编辑器自动滚动优化

拖拽选择时滚动速度不再受帧率影响，靠近边缘时速度约每秒一视口，远离时加速。

03

Markdown 链接验证

启用 `markdown.validate.enabled` 后可自动检测无效的本地文件、图片及标题链接，提前发现断裂引用。

04

终端 Quick Fixes

对常见命令错误（如误拼 Git 命令、未设置上游）和端口冲突，提供一键修正建议，通过 `Ctrl+.` 触发。

05

嵌套 Git 仓库支持

自动发现并显示 Git 子模块及嵌套仓库（默认扫描深度1级，可调 `git.repositoryScanMaxDepth`）。

06

GitHub Enterprise Server 认证改进

登录流程与 github.com 一致，不再需要手动创建个人访问令牌 (PAT)。

Editor

工作台与编辑器

  • 编辑器悬停体验改善：鼠标移向悬停区域时保持可见，不再轻易消失，方便选取快速修复项。可设 editor.hover.sticky: false 恢复旧行为。
  • Code Action 新分组与样式：新增 Inline 和 Move 分组，控件改用 editorWidget.* 主题色，更统一。
  • 树视图查找控件垂直移动：按 ⌥⌘F 后可将查找控件上下拖动，避免遮挡元素；关闭后记住上次搜索词。
  • 媒体文件内置预览：支持 WAV、MP3、Ogg、FLAC 音频及 H.264、VP8 视频（注意 AAC 音频无声）。
  • 文件资源管理器增量命名可选禁用：设置 explorer.incrementalNaming: disabled 后粘贴重复文件名直接提示覆盖，类似原生资源管理器。
  • 编辑 VS Code 应用文件夹内文件时警告：防止意外修改安装目录中的文件。
  • 合并编辑器设计调整：默认关闭（git.mergeEditor: false），冲突文件支持“在合并编辑器中解决”入口，改用 CodeLens 文本标签。
Terminal

源代码管理与终端

  • SSH 密钥密码支持：Git 操作使用带密码的 SSH 密钥时，弹出输入框输入密码（不缓存）。
  • 切换分支前拉取：启用 git.pullBeforeCheckout 后，若无待推送提交，会在切换前自动快速拉取。
  • 多远程仓库拉取选择：执行 fetch 时显示 Quick Pick 列表，可选单个远程或全部。
  • 中止合并命令：新增 Git: Abort Merge 命令。
  • 终端超链接支持：应用发出的超链接显示为标签（虚线下划线），通过转义序列实现。
  • Shell 集成改进：修复 bash $PREFIX 冲突、zsh $ZDOTDIR 边缘情况，多用户共享机器更好处理。
  • 终端音频铃声：通过 terminal.integrated.enableBell 启用。
Extension

语言与扩展

  • Markdown 提取链接定义与整理：支持将内联链接提取为引用链接定义，并自动整理所有定义到文件末尾、排序、移除未使用。
  • CSS @property / @layer 支持：语言服务现已识别这两种新的 at-rule。
  • 扩展视图优化：最近更新（7天内）和需要关注的扩展（待更新、待重载）优先显示，活动栏徽章显示数量。
  • 扩展忽略更新：点击“忽略更新”按钮可从关注列表移除，也支持安装特定版本后自动忽略。
  • 已安装扩展排序：可按安装数、评分、名称、发布日期、更新日期排序。
  • 任务固定与完成提示音：常用任务可固定到列表顶端；任务完成时播放音频（audioCues.taskCompleted）。
  • 笔记本 JavaScript 输出模块化：变量不再泄漏到其他输出，并支持 import。
  • 网页版：浏览器扩展快速打开仓库：安装 Chrome/Edge 扩展后，地址栏输入 code <仓库名> 即可打开 vscode.dev。
  • 跨环境携带未提交修改：使用“继续工作”命令时，可通过 Edit Sessions 将未提交修改自动带到 codespace、本地克隆等目标环境。
