03

Confirm edits to sensitive files

04

Agent 模式下的变更文件列表默认折叠，显示增减行数；暂存或提交后自动接受变更，减少视觉干扰。

06

Use AGENTS.md to add chat context

工作区根目录的 AGENTS.md 文件自动作为聊天上下文，支持团队为不同 AI 代理提供统一指令。

CHAT

Chat / AI Copilot

  • Custom chat modes in prompt files：Prompt 文件现可引用自定义聊天模式，而不仅限于内置的 agent/edit/ask。
  • Configure prompt file suggestions (Experimental)：新设置 chat.promptFilesRecommendations 支持 when-clause 表达式，按文件类型或上下文动态推荐 prompt 文件。
  • Select tools in tool sets：工具选择器显示每个工具集内的工具，并允许单独启用/禁用。
  • Configure font used in chat：新增 chat.fontFamily 与 chat.fontSize 设置，可独立配置聊天消息字体。
  • Math rendering enabled by default：Chat 响应中的数学方程（$...$ 和 $$...$$）默认通过 KaTeX 渲染为格式化公式。
  • Collaborate with coding agents (Experimental)：新增 Chat Sessions 视图，支持多代理并行管理、状态栏监控、与 GitHub 代理无缝衔接。
  • Semantic workspace search improvements：#codebase 工具改用新 embedding 模型，搜索结果更佳，存储空间降至原来的 6%。
TERMINAL

Terminal

  • Terminal window discoverability：新增 ⌃⇧⌥` 快捷键及多个菜单入口创建独立终端窗口，并自动进入紧凑模式。
  • Terminal actions in terminal editors：编辑器区域和终端窗口中的终端现在也显示新终端、清除等操作按钮，支持右键调整。
  • Terminal IntelliSense improvements (Preview)：性能提升，git 补全添加图标，新增大量命令补全 specs（如 adb, docker, gh, jq 等）。
  • Terminal sticky scroll enabled by default：终端粘性滚动默认开启，兼容 editor.tabFocusMode，并改进了与分页器的搭配。
  • Configurable terminal tool shell integration timeout：runInTerminal 工具的 shell 集成等待超时现可通过 chat.tools.terminal.shellIntegrationTimeout 设置。
EDITOR

Editor & Source Control

  • Window border color on Windows：新增 window.border 设置，支持主题色、系统色或自定义颜色，可区分不同工作区窗口。
  • Editor tab index：启用 workbench.editor.showTabIndex 后，标签页标题显示索引编号，便于键盘导航。
  • Editor tab bar scrollbar visibility：workbench.editor.titleScrollbarVisibility 控制标签栏滚动条的显示策略（auto/visible/hidden）。
