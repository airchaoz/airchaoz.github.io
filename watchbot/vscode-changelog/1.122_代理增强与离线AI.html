核心亮点

01

1M token 上下文窗口

支持 Anthropic 和 OpenAI 模型的百万级上下文窗口，处理更大代码库和更长对话，但注意消耗更多AI积分。

02

离线 BYOK（无需登录）

无需 GitHub 登录即可使用自有语言模型 API 密钥，在气隙或离线环境中启用聊天、工具和 MCP 服务器，完全本地工作流成为可能。

03

集成浏览器设备模拟

在 VS Code 内置浏览器中直接模拟不同设备的屏幕尺寸、触摸行为及 User-Agent，无需切换外部工具即可调试响应式布局。

04

改进的 Issue 报告向导

开启后引导用户创建包含截图和录屏的高质量 Issue 报告，帮助开发者更精准地反馈问题。

05

代理窗口（Preview）重大改进

新增悬浮详情查看会话信息，Insiders 版支持本地 harness 和自定义代理选择器，强化跨项目代理工作流。

06

终端沙箱行为优化

沙箱模式现在仅适用于 Default Approvals 模式，避免在无审批流程中产生混淆，平衡安全性和可用性。

重点功能

重点功能速览

  • GitHub Copilot 计费模型变更：Copilot 转为基于使用量的计费，每次交互消耗 AI 积分，状态仪表盘展示积分消耗，模型选择器显示各模型成本信息，帮助开发者合理选择模型。
  • 代理窗口与 OpenTelemetry：代理会话列表新增悬浮详情；Insiders 中本地 harness 可选；本地代理会话发出标准化的 OpenTelemetry 信号（属性命名空间 github.copilot.*），便于监控代理使用。
  • 语言模型增强：1M 上下文窗口支持；BYOK 无需登录；离线下 utility 模型通知引导配置；自定义端点提供者稳定可用；可在代理窗口中直接管理语言模型，并新增细粒度操作（更新 API Key、添加模型、重命名组等）。
  • 集成浏览器新能力：设备模拟工具栏（通过溢出菜单打开），支持自定义 User-Agent；新增“添加截图到聊天”功能，将当前浏览器视口截图作为聊天上下文，便于调试 UI 问题。
  • 编辑器体验改进：新增 Issue 报告向导（设置 issueReporter.wizard.enabled 启用），引导用户添加详细描述、截图和录屏，提升报告质量。
