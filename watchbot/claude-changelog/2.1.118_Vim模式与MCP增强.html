核心亮点

01

Vim 可视化模式

新增 `v`（字符选）和 `V`（行选）两种可视化模式，支持选区操作和视觉反馈，大幅提升资深用户编辑效率。

02

Hooks 直接调用 MCP 工具

Hooks 配置新增 `type: "mcp_tool"`，允许在生命周期钩子中直接调用 MCP 工具，扩展插件与自动化能力。

03

自定义主题系统

通过 `/theme` 创建并切换命名主题，支持 JSON 文件或插件 themes 目录分发，满足个性化与团队共享需求。

04

禁止更新控制增强

新增 `DISABLE_UPDATES` 环境变量，完全阻断所有更新路径（包括手动 `claude update`），比 `DISABLE_AUTOUPDATER` 更严格。

05

Auto mode 允许/拒绝列表扩展

支持在 `autoMode.allow`、`autoMode.soft_deny` 或 `autoMode.environment` 中使用 `"$defaults"` 字段，在内置规则基础上追加自定义规则。

06

MCP OAuth 与认证修复

修复了多个 OAuth token 过期、并发竞争、密钥链冲突及 headersHelper 认证状态卡死问题，提升 MCP 连接安全性与稳定性。

Editor

编辑器与用户界面

  • 命令合并：/cost 和 /stats 合并为 /usage，两个旧命令保留为快捷入口直接打开对应标签。
  • 颜色同步：/color 设置的主题色在连接 Remote Control 时同步到 claude.ai/code。
  • 模型选择器改进：当使用自定义 ANTHROPIC_BASE_URL 网关时，/model 选择器现在会正确读取 ANTHROPIC_DEFAULT_*_MODEL_NAME/_DESCRIPTION 环境变量覆盖。
  • UI 修复：修复了“新消息”滚动按钮和 /plugin 徽章文本不可读的问题；修复了 Alt+K/X/^/\ 导致键盘输入冻结的 bug。
MCP

MCP 与扩展

  • 插件标签命令：新增 claude plugin tag 命令，可为插件创建带版本验证的 release Git 标签。
  • 依赖解析修复：plugin install 对已安装插件不再跳过依赖版本重解析，避免错误版本残留。
  • MCP 认证流程修复：修复了 OAuth token 响应中缺少 expires_in 时每小时重认证的问题；修复了 insufficient_scope 403 在 token 已有该 scope 时静默刷新而非提示重新授权的问题。
  • OAuth 并发与密钥链修复：修复了并发刷新 OAuth token 时竞态条件覆盖新 token 导致需重新 /login 的问题；修复了 Linux/Windows 上凭据文件损坏的问题。
  • 环境 token 登录修复：使用 CLAUDE_CODE_OAUTH_TOKEN 启动会话后执行 /login 不再无效，环境变量 token 会被清除以便磁盘凭据生效。
其他修复

其他修复与改进

  • 会话恢复改进：--continue/--resume 现在能发现通过 /add-dir 添加了当前目录的会话。
  • Fork 性能优化：/fork 不再将整个父会话写入磁盘，改为写入指针并在读取时按需加载，显著减少写入量。
  • Remote Control 稳定性：修复了 JWT 刷新期间因短暂 CCR 初始化波动导致会话被归档的问题。
  • 子代理工作目录恢复：修复了通过 SendMessage 恢复的子代理未保持其原始 cwd 的问题。
  • Agent hooks 过滤：agent 类型 hooks 现在只在 Stop 或 SubagentStop 事件时触发，避免在其他事件中因缺少消息而失败。
