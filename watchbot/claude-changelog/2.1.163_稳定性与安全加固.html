核心亮点

01

受管版本范围强制

新增 `requiredMinimumVersion` / `requiredMaximumVersion` managed settings，Claude Code 版本超出范围时拒绝启动并引导用户使用批准版本，便于组织统一管控。

02

后台命令不再阻塞 `claude -p`

修复了 `claude -p` 在后台命令永不退出时无限挂起的问题：stdin 关闭后约 5 秒自动停止后台 shell。

03

Hooks 子代理反馈无错标

Stop 与 SubagentStop hooks 的 `hookSpecificOutput.additionalContext` 现在可向 Claude 传递反馈且不被标记为 hook 错误，保持对话继续。

04

Skills 字面量 `$` 转义

新增 `\$` 转义语法，允许在 command body 中包含字面量 `$` 后跟数字，解决了书写密码或环境变量时的冲突。

05

MCP 服务器会话 ID 一致

stdio MCP 服务器在 `--resume` 时现在接收与 hooks/Bash 相同的 `CLAUDE_CODE_SESSION_ID`，确保可观测性链路完整。

06

Deny 规则覆盖 `$HOME` 引用

修复了主目录路径 deny 规则（如 `Read(~/Desktop/**)`）未阻止通过 `$HOME` 引用的 Bash 命令，强化了安全边界。

CLI

CLI 与插件管理

  • 插件列表命令：新增 /plugin list 命令，支持 --enabled/--disabled 过滤器，方便查看已安装插件状态。
  • /btw 复制快捷键：/btw 回复右下角新增 "c to copy" 快捷键，复制原始 markdown 到剪贴板，保留格式。
  • 后台会话无缝升级：Background agent 会话现在会在后台自动更新至新 Claude Code 版本，重新打开时无需冷重启。
  • 命令与技能菜单更清晰：内置命令和技能在 / 菜单中的描述更加清晰易懂。
  • 订阅切换提示位置优化：订阅切换建议从 toast 改为启动公告位展示，减少干扰。
Terminal

终端与后台稳定性

  • 退出 agent 视图不再卡死：修复了按 Esc 退出 agent 视图时终端错位和数秒挂起的问题。
  • 粘贴操作后键盘恢复正常：修复了因终端丢弃粘贴结束标记导致键盘永久无响应的问题。
  • 桌面端停止按钮状态同步：修复了后台任务 chip 上的 Stop 按钮点击后，若底层进程已消失，chip 未及时清除的问题。
  • 对话框关闭后无残余文本：修复了关闭 /mcp、/plugins 等面板对话框后，会话记录中遗留 "(no content)" 行的问题。
  • Bash() 条件匹配子 shell：hook if: "Bash(...)" 条件现在能正确匹配包含 $() 或反引号的子 shell 命令，而非每次触发。
