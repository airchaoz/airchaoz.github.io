核心亮点

01

子代理错误处理增强

子代理被速率限制或服务器错误中断时，不再静默失败，而是将部分工作返回给父代理；API 错误（如用量超限）也会如实报告给父代理。

02

后台 Agent 守护进程崩溃修复

修复 Linux 上后台 agent 的守护进程在非正常关闭后每约 50 秒自杀并杀死所有运行中 agent 的问题，以及 macOS 上 SSH 后台 agent 无法冷启动的回归问题。

03

SSL 证书错误即时提示

TLS 检查代理、证书过期等 SSL 错误不再重试消耗配额，而是立即失败并显示修复指导。

04

流式响应中断保留

API 在中途返回过载/服务器错误时，不再丢弃已输出的部分，而是保留并附带“不完整响应”提示。

05

瞬态 429 自动重试

非用量限制的临时速率限制（429）对订阅用户自动退避重试，无需手动介入。

子代理与后台会话

  • Stacked Slash-Skill 多技能加载：连续斜杠技能调用（如 /skill-a /skill-b do XYZ）现在会加载所有前置技能（最多 5 个），不再仅加载第一个。
  • 闲置子代理折叠显示：当其他子代理仍在工作时，闲置子代理不再从 agent 面板消失，而是折叠成一个可展开的摘要行。
  • 后台任务进度条停滞修复：后台 job 在运行长命令时进度指示器不再停滞数分钟。
  • 低内存提示：内存紧张的机器上后台 session 不再显示泛泛的错误，而是提示低内存并建议释放资源。
  • 远程会话状态闪烁修复：后台 agent 完成时，远程会话在 Working 和 Idle 之间快速闪烁的问题已修复。
  • 后台 Agent 重入与停止修复：修复 claude stop 与后台 agent 重入的竞态条件，重入现尊重停止命令；修复 claude --dangerously-skip-permissions daemon <subcommand> 被错误当作聊天提示的问题。
错误处理

错误处理与重试改进

  • Hook 错误暴露：SessionStart、Setup、SubagentStart 等 Hook 退出码为 2 时，stderr 不再静默隐藏，错误信息现直接显示在 transcript 中。
  • 重试计数提升：CLAUDE_CODE_RETRY_WATCHDOG 环境变量将默认非容量性临时错误的重试次数提升至 300，同时取消 CLAUDE_CODE_MAX_RETRIES 的 15 次上限。
  • SendMessage 误路由检测：当重新生成的 agent 重用之前 agent 的名称时，SendMessage 静默错误路由问题已被检测并提示调用者重新定向。
