TL;DR

核心亮点

01

配置持久化（Settings Persist）

`/config` 设置现在保存到 `~/.claude/settings.json`，并参与项目/本地/策略覆盖优先级，解决重启后设置丢失的问题。

02

多平台PR链接支持

`--from-pr` 新增支持 GitLab Merge Request、Bitbucket Pull Request 和 GitHub Enterprise PR URL，扩展跨平台集成。

03

权限模式完善

`--print` 模式现在遵守 agent 定义的 tools/disallowedTools，PowerShell 命令可自动批准，`--agent` 支持内置 agent 的 permissionMode。

04

MCP服务器并行重连

Subagent 和 SDK MCP 服务器重新配置时改为并行连接，显著提升初始化速度。

05

OpenTelemetry 增强

`tool_result` 和 `tool_decision` 事件新增 `tool_use_id` 和 `tool_input_size_bytes`，支持更精细的工具调用追踪。

06

Vim模式修复

Esc 在 INSERT 模式下不再将队列中的消息拉回输入框，需要再按一次 Esc 才能中断，修复编辑行为。

设置与配

设置与配置

  • prUrlTemplate 设置：新增 prUrlTemplate 配置项，允许将底部 PR 徽标指向自定义代码审查 URL，默认 github.com。
  • CLAUDE_CODE_HIDE_CWD 环境变量：设置后隐藏启动标志中的工作目录，适用于隐私或少干扰场景。
  • /config 设置持久化：主题、编辑器模式、verbose 等设置现在持久化到 ~/.claude/settings.json，并参与覆盖优先级。
Security

权限与安全

  • blockedMarketplaces 修复：安全配置 blockedMarketplaces 现在正确强制执行 hostPattern 和 pathPattern。
  • 工具权限一致性：--print 模式与交互模式一致，遵守 agent 的 tools/disallowedTools 前端定义。
  • PowerShell 自动批准：PowerShell 工具命令在权限模式下可自动批准，与 Bash 行为一致。
  • 内置 agent permissionMode：--agent <name> 现在能正确应用 agent 定义中的 permissionMode。

  • MCP HTTP OAuth 修复：修复 OAuth 发现请求返回非 JSON 正文时“Invalid OAuth error response”错误；修复 --client-secret 凭证未发送问题。
  • MCP 头变量替换：HTTP/SSE/WebSocket MCP 服务器的 headers 中 ${ENV_VAR} 占位符现在正确替换。
  • 插件 MCP 服务器修复：修复 Windows 上插件缓存不完整导致服务器不启动的问题；修复 ${user_config.*} 引用可选空字段时的失败。

  • 工具消失回归修复：修复在原生 macOS/Linux 构建中，当 Bash 工具被权限拒绝时 Glob 和 Grep 工具消失的问题。
  • 全屏滚动修复：修复全屏模式下向上滚动后每次工具完成时自动弹回底部。
