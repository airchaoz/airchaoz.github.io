TL;DR

核心亮点

01

MCP 安全加固：阻止自批准服务器自动启动

`claude mcp list`/`get` 不再执行仓库通过承诺的 `.claude/settings.json` 自批准的 `.mcp.json` 服务器；不可信工作区显示「⏸ Pending approval」，防止恶意代码自动运行。

02

后台会话与代理可靠性飞跃

长时间运行的命令和工作流在进程停止、重启或更新后仍能存活，Windows 后台 shell 被移交而非杀死；被守护进程重启杀死的后台代理自动恢复，不再丢失进度。

03

组织默认模型支持

管理员可在组织控制台设置默认模型，用户在 `/model` 中看到「Org default」/「Role default」标识，无需手动选择即使用组织统一模型。

04

性能与效率优化

`/code-review` 合并五个清理查找器，token 使用量降低约 25%；终端 UI 跳过无副作用的子树遍历减少每帧渲染工作。

05

流空闲看门狗默认开启

所有提供商默认启用 5 分钟无事件中止并重试机制（`CLAUDE_ENABLE_STREAM_WATCHDOG=0` 可关闭），防止响应卡死。

Security

安全与权限

  • MCP 自批准服务器不再自动执行：claude mcp list/get 忽略仓库提交的 .claude/settings.json 中自批准的 .mcp.json 服务器，不可信工作区强制审批。
  • --dangerously-skip-permissions 修复：该标志不再悄悄回退到 auto 模式，而是正确显示免责声明并对衍生代理应用绕过模式。
  • 远程控制条件禁用：当 ANTHROPIC_BASE_URL 指向非 Anthropic 主机时（类似 Bedrock/Vertex 场景），Remote Control 自动禁用。
后台会话

后台会话与代理

  • 后台任务进程存活：停止、重启或更新 Claude Code 进程后，后台会话中的长命令和工作流自动延续；Windows 上后台 shell 被移交而非杀死。
  • 后台代理自动恢复：守护进程重启导致的后台代理 worker 被杀，下次打开 agents 视图时自动从断点恢复。
  • 修复唤醒后台作业删除对话：transcript 探测误读真实对话时，不再永久删除对话并重跑原始提示，而是将文件搁置。
  • agents 侧边栏多项修复：修复键盘焦点卡死、后台作业丢失子代理类型、会话状态显示不正确等问题；完成的行不再在「Done」和「Needs your input」间闪烁，停滞的代理标记为「Needs attention」。
Perf

用户界面与性能

  • 可读的会话默认名称：新会话启动时自动赋予易识别的名称，便于在列表中定位和消息。
  • 可点击的文件附件：聊天中的文件附件支持 Cmd/Ctrl+点击，在 Finder/Explorer 中直接定位文件。
  • /code-review token 减少 25%：合并五个清理查找器，每次 code review 平均节省约 25% token 消耗。
  • 终端 UI 渲染优化：流式渲染时跳过无副作用的子树遍历，减少每帧计算量，提升流畅度。
  • 从前景会话打开 agents 视图简化：仅需按一次 ← 键（之前需两次），与后台会话行为一致。
  • MCP OAuth 作用域修复：当未指定 scope 时，不再请求授权服务器完整的 scopes_supported 列表，修复 GitLab 自托管等企业 IdP 的 invalid_scope 错误。
