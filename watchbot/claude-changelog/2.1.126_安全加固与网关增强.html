核心亮点

01

网关模型列表集成

当 `ANTHROPIC_BASE_URL` 指向兼容网关时，`/model` 选择器可自动列出网关 `/v1/models` 端点提供的模型，简化自定义模型切换。

02

项目状态清理命令

新增 `claude project purge [path]` 命令，支持 `--dry-run`、`-y`、`-i`、`--all`，可删除转录、任务、文件历史及配置条目，帮助管理项目生命周期。

03

强制跳过权限的路径扩展

`--dangerously-skip-permissions` 现在会绕过对 `.claude/`、`.git/`、`.vscode/`、shell 配置文件等受保护路径的写入提示（灾难性删除仍保留安全兜底），提升自动化脚本可控性。

04

OAuth 粘贴登录支持

`claude auth login` 允许在浏览器回调无法到达 localhost（如 WSL2、SSH、容器）时手动粘贴 OAuth code，解决远程环境认证困难。

05

图像粘贴过大导致会话中断修复

粘贴大于 2000px 的图像时自动降采样，并自动移除历史中的超大图片后重试请求，避免会话死锁。

06

安全配置忽略修复

修复了当更高级的管理设置源缺少 `sandbox` 块时，`allowManagedDomainsOnly` / `allowManagedReadPathsOnly` 被忽略的问题，确保安全策略强制生效。

Security

安全与权限

  • 受保护路径写入绕过控制：--dangerously-skip-permissions 行为扩展至 .claude/、.git/、.vscode/、shell 配置文件等，仍有灾难删除兜底。
  • 管理设置安全策略强制：修复了多级管理设置源合并时 allowManagedDomainsOnly 等被忽略的缺陷，确保安全限制生效。
  • Windows 剪贴板安全：剪贴板写入不再将复制内容暴露在进程命令行参数中（避免 EDR/SIEM 监控泄漏），并修复了 >22KB 选择无法粘贴的问题。
项目管理

项目管理与认证

  • 项目状态清除：claude project purge 支持删除项目所有状态，包含 --dry-run、-y、-i、--all 选项。
  • OAuth 远程登录：claude auth login 支持手动粘贴 OAuth code，解决 WSL2/SSH/容器等环境回调失败。
  • 组织授权错误提示改进：当组织不允许 OAuth 时，不再显示登录界面，而是提示联系管理员，减少困惑。
