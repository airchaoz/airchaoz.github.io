核心亮点

01

/fork 重构为后台会话复制，子代理改用 /subtask

/fork 命令不再启动子代理，而是将当前对话复制到一个新的后台会话（独立显示在 claude agents 中），原有子代理功能由新命令 /subtask 替代。

02

新增会话级资源上限，防止失控循环

引入 CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION 和 CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION 环境变量（默认均为200），限制 Web 搜索和子代理的滥用；/clear 会重置预算。

03

MCP 工具超时自动后台化

MCP 工具调用运行超过2分钟自动移入后台，保持会话响应；可通过 CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS 配置阈值或关闭。

04

/resume 支持从历史会话拾取器恢复

在 agent 视图中键入 /resume 将打开一个包含过往会话（包括已删除会话）的拾取器，选择后作为后台会话恢复。

05

修复计划模式下自动执行文件修改命令绕过权限提示

plan mode 不再允许自动运行如 touch、rm 等会修改文件的 Bash 命令，现在会触发权限提示或 SDK 的 canUseTool 回调。

06

Prompt 缓存改进，支持网关与自定义 Base URL

对话中期的 system prompt 块现在也可被缓存（适用于 Bedrock、Vertex 等网关），减少 token 消耗。

Security

权限与安全

  • 修复工作树创建跟随 symlink 导致文件外泄：修正在 .claude/worktrees 中使用 repo 提交的 symlink 时，工作树创建可能将文件写到仓库外部的问题。
  • 修复 continue:false hook 的 halt 被丢弃：当工具失败或完成时，continue:false 钩子的 halt 不再被丢弃；钩子基础设施错误现在正确报告而非误报为用户拒绝。
  • 修复 SIGTERM 导致 Bash 命令进程树孤岛：在 print/SDK 模式下，Bash 工具运行时收到 SIGTERM 不再遗留进程树；CLI 现在会中止回合、杀死进程树并以退出码 143 退出。
  • 修复 auto-mode 拒绝通知截断emoji导致乱码：自动模式拒绝原因过长被截断时，不再因截断中emoji而产生损坏字符。
  • 修复 hosted sessions 忽略传输设置：当仓库配置了 mTLS 证书、extra CA bundles 或 OAuth scopes 时，hosted sessions 启动不再失败；这些传输设置现在被忽略并给出警告。
AGENT

Agent协作与工具

  • 减少 inter-agent 消息 token 消耗：在子代理通信中，SendMessage 的内容不再在回放历史和工具结果中重复，降低 token 使用。
  • 修复 WebSearch/WebFetch 返回 'API Error' 文本：当 API 过载时，web search 和 web fetch 不再返回 'API Error' 文本作为搜索结果或页面内容，而是更合理的处理。
  • 改善搜索可靠性，重试 529 和限速：web search 和 web fetch 增加对 529 错误和速率限制的重试，带有有界退避。
  • 修复 OTLP 事件日志缺少 trace_id/span_id：在 SDK/headless 模式下，当设置 TRACEPARENT 时，OTLP 事件日志记录现在包含正确的 trace_id 和 span_id。
  • 修复 OpenTelemetry HTTP 导出被 Azure Monitor 拒绝：OpenTelemetry HTTP 导出不再使用 chunked transfer encoding，解决 411/400 错误。
  • 修复大量图片导致 'Request too large' 错误：对话中包含大量图片时不再错误触发 'Request too large'，改进错误信息以说明实际原因。
  • 修复 /ultrareview 多项问题：修复 PR 引用格式（#123, PR 123, URL）被拒绝、分支远程拉取失败、账单确认跳过、非 git 目录错误提示等。
  • Task tool 的 mode 参数废弃：子代理现在默认继承父会话的权限模式，Task tool 的 mode 参数被忽略（已废弃）。

其他改进与修复

  • 修复 Ctrl+J 换行问题：在支持 extended key reporting 的终端上，Ctrl+J 现在能插入换行，并在 ? 帮助覆盖层中显示换行快捷键。
  • 修复 diff 预览在窄布局丢失行号：窄布局下 diff 预览不再丢失行号和 +/- 标记。

  • 修复 background 会话重新打开失败：从 agent 视图重新打开停止的后台会话不再静默失败；现在会尝试恢复，或显示原因并允许强制重启。

  • 修复 'File has not been read yet' 错误：在恢复会话之前，如果曾经用 offset/limit 读取文件，编辑时不再出现此错误。
  • 变更会话transcript记录推理努力级别：会话转录现在记录每个 assistant 消息的推理努力级别，增强可观测性。

  • 修复 streaming control request 标记完成过早：流控件请求现在等待处理器完成后才标记完成，避免会话重启后请求丢失。
